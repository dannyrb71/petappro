Business-logic tokens

Domain semantics — booking, payment, service, notification, capacity, risk, role, pet-gender. Theme-independent: they encode meaning, not brand, so they do not live in the theme modes. Atoms below consume them.

02 · Atom

Button

SpecCanonical Button
OBJECTIVE
Create the canonical Button for the Petappro Design System.

REQUIREMENTS
- Variants: Primary, Secondary, Tertiary/Ghost, Destructive, Success (booking-green — booking-CREATION actions only).
- Sizes: Lg 52 / Md 44 / Sm 36 / Xs 28 px height; Md default; ≥44 is the standard tap target.
- Icon slots: optional leading + trailing (Lucide, 1.9 stroke), sized to the label.
- States: default, hover, pressed, focus-visible (3px ring ≥3:1), disabled, loading (spinner replaces label, width locked).
- Tokens: fill/label from theme color roles; radius = comp.button.shape (fully rounded); gap = space/2; height per size token.
- Auto Layout: horizontal, hug, center; padding from space tokens.
- RN compatibility: props (variant, size, leadingIcon, trailingIcon, state, label) named to map onto RN.

ACCEPTANCE CRITERIA
- No literal hex; all variants use shared tokens. No detached instances.
- Meets WCAG AA (contrast + focus ring + 44px default target).
- Minimal, well-named component properties. One reusable component.

DOCUMENTATION
- Purpose · Usage · When not to use · Variant descriptions (booking-creation = Success) · A11y · Dev notes.

03 · Input / Field

SpecText field / input
OBJECTIVE
Create the canonical Field (text input) for Petappro.

REQUIREMENTS
- Anatomy (slots): label · input · optional leading icon · optional trailing icon/affix (e.g. "$") · hint/help · error message.
- Types: text, number, date, textarea (multi-line), select. Share one visual shell.
- States: default, focus (border = primary + 3px focus ring), filled, disabled, error (error border + message), read-only.
- Tokens: container = comp.input.container; border = comp.input.outline / focus; radius = comp.input.shape (12); text = on-surface; label = body-sm 600; hint = label size, on-surface-variant; error = business error/notif-urgent.on.
- Sizing: min height 44; padding from space tokens; textarea min 92.
- Auto Layout: vertical group (label/input/hint), gap = space/2.
- RN compatibility: props (label, type, value, placeholder, leadingIcon, trailingAffix, state, hint, error).

ACCEPTANCE CRITERIA
- No literal hex. Error state never signals by color alone (icon + message). Label always present (or aria-label noted).
- Focus ring ≥3:1; 44px min target. One shell, variants for type.

DOCUMENTATION
- Purpose · Usage · When not to use · Type/state descriptions · A11y (label association) · Dev notes.

04 · Avatar

Generic avatar atom. The Pet Avatar domain component (gender ring, puppy badge) and the Multi-pet Stack build on top of this — noted for Part 4.

SpecAvatar
OBJECTIVE
Create the canonical Avatar for Petappro (people and, as a base, pets).

REQUIREMENTS
- Content: image, or fallback (monogram initials, or a Lucide glyph e.g. dog).
- Shape: circle (default); size variants Xs 24 / Sm 32 / Md 40 / Lg 64 / Xl 96.
- Optional ring: none (default) | brand | gender (male #4882AF / female #AB5FB9 via business tokens) — for the Pet Avatar extension.
- Optional status dot (online/active) bottom-right, and an optional corner badge slot (e.g. puppy) — expose as boolean/slot, off by default.
- Tokens: fallback bg = surface-container-high; monogram text = on-surface-variant; ring color from theme or business pet-gender tokens; border 2px.
- Auto Layout / constraints: image fills; badge pinned bottom-right.
- RN compatibility: props (src, name, size, ring, statusDot, badge).

ACCEPTANCE CRITERIA
- No literal hex (gender ring uses business tokens). Fallback always legible (AA). Fixed square ratio at every size.
- Base atom stays generic; pet-specific styling comes from the ring/badge props, not a fork.

DOCUMENTATION
- Purpose · Usage · When not to use · Size/ring guidance · A11y (alt text) · Dev notes.

05 · Icon

SpecIcon
OBJECTIVE
Establish the Icon atom + the Petappro icon set.

REQUIREMENTS
- Library: Lucide. Stroke 1.9, round caps/joins, 24px grid.
- Sizes: 16 / 18 / 20 / 24 (default) / 28, as size variants; stroke stays optically consistent.
- Color: inherits currentColor by default; may be set to any theme or business token.
- Curated set (import as components): navigation (house, calendar-days, dog, menu, search, bell, user-round, settings), services (moon, sun, footprints, house-plus, scissors, graduation-cap, car-front, bone), status (shield-check, circle-check-big, lock, clock, handshake, calendar-check, clipboard-list, credit-card), care (syringe, stethoscope, pill, utensils, camera, map-pin, triangle-alert, notebook-pen), actions (plus, chevron-right, pencil, x, check, message-circle, star, share).

ACCEPTANCE CRITERIA
- Single Icon component with a name/size property; no ad-hoc pasted SVGs elsewhere. Consistent stroke + grid.

DOCUMENTATION
- Purpose · Usage · Naming map (semantic → glyph) · Sizing · Dev notes (icon font vs SVG in RN).

06 · Money

SpecMoney
OBJECTIVE
Create the Money atom — the single way an amount is rendered.

REQUIREMENTS
- Formats value as currency (2 decimals, locale-aware; USD default).
- States: default; cancelled = strikethrough + on-surface-variant color (matches live app rule — struck everywhere a cancelled amount appears).
- Optional emphasis: regular vs bold (totals).
- Tokens: text = on-surface / on-surface-variant; no literal colors.
- RN compatibility: props (value, cancelled, emphasis, currency).

ACCEPTANCE CRITERIA
- Strike + mute always travel together for cancelled. No hard-coded color. Never re-implement a struck price inline.

DOCUMENTATION
- Purpose · Usage · When not to use · Dev notes (formatting util shared with pricing engine).

07 · Status & Payment badge

Upcoming In progress Completed Cancelled Unpaid
SpecStatus / Payment badge
OBJECTIVE
Create the canonical Badge for lifecycle + payment status (the app's single source for status pills).

REQUIREMENTS
- Booking statuses: Upcoming, In Progress, Completed, Cancelled.
- Payment statuses: Paid, Unpaid, Partial, Overridden.
- Skin: soft container bg + accessible on-container text from the BUSINESS tokens (booking.* / payment.*). Cancelled may add a subtle outline.
- Never color-alone: each carries a label (and optional leading icon/dot).
- Shape: comp.badge.shape (fully rounded); label = 11–12px 700; padding from space tokens.
- RN compatibility: props (kind[booking|payment], status, label?, icon?).

ACCEPTANCE CRITERIA
- Colors come only from business tokens; theme-independent. Text ≥4.5:1 on its container. Label always present. One component, status as a property (no per-status forks).

DOCUMENTATION
- Purpose · Usage · Status → token map · When not to use (don't use for services — use Service Pill) · A11y · Dev notes.

08 · Service pill

SpecService pill
OBJECTIVE
Create the canonical Service Pill — one source for every service/activity tag.

REQUIREMENTS
- Services: Boarding, Daycare, Walk, In-home Sitting, Grooming, Training, Pickup/Dropoff, Meet & Greet.
- Skin: solid fill = business service token, white label + leading icon — EXCEPT Meet & Greet, the only OUTLINED pill (white bg, colored stroke + text) per the live app.
- Selectable variant (for booking flows): unselected (outline/neutral) vs selected (filled brand). Expose "selectable" + "selected" props.
- Shape: comp.chip.shape (fully rounded); height ~22–28; icon = Lucide.
- RN compatibility: props (service, label?, selectable, selected, icon).

ACCEPTANCE CRITERIA
- Colors from business service tokens only. Solid + outlined pills align on the same baseline/height. Meet & Greet is the sole outlined case. One component; service as a property.

DOCUMENTATION
- Purpose · Usage · Service → token/icon map · Solid vs outlined rule · A11y · Dev notes.

09 · Switch

SpecSwitch / toggle
OBJECTIVE
Create the canonical Switch (binary toggle) for Petappro.

REQUIREMENTS
- States: off, on, disabled-off, disabled-on, focus-visible.
- On = comp.switch.on (theme secondary-solid); track off = outline-strong; thumb = white with elevation-1.
- Motion: thumb slides medium + standard easing; honor reduced-motion.
- Size: track 46×28, thumb 22 (≥44px hit area via padding).
- RN compatibility: props (checked, disabled, onChange, label?).

ACCEPTANCE CRITERIA
- Token-driven colors; 44px effective target; visible focus ring; clear on/off beyond color (thumb position).

DOCUMENTATION
- Purpose · Usage · When to use vs checkbox · A11y · Dev notes.
